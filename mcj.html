My College Journey

A mobile application that aids college students in tracking their academic journey through a user-friendly design that allows them to add terms, courses, assessments, notifications and notes.

  • View Project

Tools Used: Android Studio, SQLite, GitHub


My College Journey is an application for the average college student to track their academic progress from orientation to graduation, with simplicity and convenience in mind. Currently, the application is only available for Android phones due to their convenience, but a tablet version would provide more adequate space for the course details activity layout. In landscape mode I would move the course and assessment lists fragments to the right of the term and course details, instead of being below them. My College Journey was developed with a minimum SDK of 26, targeting an SDK of 32. While developing and debugging the application, I used an emulator to test across multiple APIs since I only own one physical phone. Once I completed the application, I ran it on my physical device to test it in real-world conditions. Although emulators are useful tools for debugging and testing across multiple devices, a physical device allows you to see how it interacts with the hardware and software for performance testing.

I began by breaking each requirement into smaller increments before I started programming the applications. As I worked through the requirements, I encountered many challenges including:

  1. As I visualized the layout of the app, I knew I would need to incorporate color into the design to aid in the navigation for a better user experience.
  2. After successfully programming all the menus, I realized that my back navigation arrow no longer worked properly.
  3. Due to the length of the course details activity, and the small size of the target device, I struggled to allow the user to scroll through the course details and the assessment list while in landscape mode.

Solutions:

  1. By color coordinating each list item and the “+” floating action button, with a unique color, I hoped to enhance the user experience by introducing a fun color scheme and a cohesive navigation system.
  2. While rewatching the first dolphin webinar, I realized that the onOptionsItemSelected method for handling the menu selection should have included a switch case for android.R.id.home .
  3. After unsuccessfully trying to wrap the recycler view for the assessment list inside the course details activity’s scrollview, I read the Android developer documentation on recycler views. (Android Developers, n.d.) Adding the scrollbars attribute and setting it to vertical, enabled the user to view the entire list of assessments added.

If I were to do the project again, I would watch the dolphin webinars while taking detailed notes on key topics. I would incorporate my color scheme into the resources' colors instead of using hexadecimal colors for easier updating. In addition, I would reference the Android developer website earlier in my troubleshooting process before searching Google. Furthermore, reducing each requirement to manageable steps and testing in increments was imperative to troubleshooting errors as I encountered them, and I would duplicate these processes. Progressing through the development of the application, I had many opportunities to further my knowledge of the Java language and learn how it interacts with the Android system.
